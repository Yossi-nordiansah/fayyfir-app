🧩 KETERANGAN PENTING STRUKTURAL

📌 1. Modular per Tahap

Setiap proses (pembelian, jemur, kupas, sortir, jual) punya folder tersendiri dengan empat jenis file utama:

input- → form data baru

list- → daftar batch / transaksi

detail- → hasil perhitungan otomatis

hapus- → aksi delete


Struktur ini memudahkan kita untuk ekspansi tahap baru di masa depan tanpa merombak struktur dasar.


---

📌 2. Reusable Component

Folder /partials/ digunakan untuk komponen layout yang sama di semua halaman — header, sidebar, navbar, dan footer.
Nantinya kita tinggal include seperti:

<?php include '../partials/header.php'; ?>
<?php include '../partials/sidebar.php'; ?>


---

📌 3. Logic Helper & Hitungan Otomatis

Semua rumus perhitungan otomatis (seperti HPP, penyusutan, margin, laba bersih) akan dikelola di:

includes/functions.php

Contoh fungsi:

function hitung_penyusutan($berat_awal, $persen_susut) {
  return $berat_awal * (1 - $persen_susut / 100);
}


---

📌 4. Koneksi Database & Konfigurasi Global

Tidak perlu membuat ulang config.php karena project ini akan menggunakan koneksi utama dari folder root:

/abdul-hadi/config.php

Jadi semua halaman di dalam belanja-harian cukup include:

require "../../config.php";


---

📌 5. Integrasi AJAX

Folder /abdul-hadi/ajax/ tetap digunakan untuk operasi dinamis seperti:

Fetch data supplier/buyer secara live

Hitung otomatis HPP sebelum simpan

Load data batch berdasarkan tahap



---

Kalau kamu setuju dengan struktur folder di atas, kita bisa lanjut ke langkah berikut:

lengkap dengan form input, tampilan kalkulasi otomatis, dan UI wireframe-nya.
